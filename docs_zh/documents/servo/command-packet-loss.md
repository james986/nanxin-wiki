# 指令丢包问题及解决方法



1. 如果一直频繁发送的话，可以会造成舵机内部muc的ram被塞满造成冗余，后续再发送的话会收不到后续的指令，处理ram里面的指令，有空间才会继续接收并执行。下图是指令在6ms导致出现丢包舵机不再运行的现象。



{{ fig_indent_adjustable("../images/image-20250722093226061.png","指令丢包问题及解决方法", "100%", "3.5em") }}



2. 为了防止指令丢包等问题，建议将指令间的间隔控制在10ms以上（极限可以到5ms）：



如：指令1–延迟10ms–指令2–延迟10ms–……
